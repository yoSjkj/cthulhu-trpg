# CoC AI 키퍼 TRPG PWA 기획안

## 프로젝트 개요

Call of Cthulhu(크툴루의 부름) 7판 룰 기반의 AI 키퍼 TRPG PWA.
1인 플레이어 + Claude AI 키퍼 구성으로 솔로 플레이를 목표로 한다.
React + localStorage + Claude API(claude-sonnet-4-20250514) 사용.

---

## 기술 스택

- Frontend: React + Vite
- 상태관리: useState / useReducer (외부 라이브러리 최소화)
- 저장: localStorage (PWA 오프라인 대응)
- AI: Claude API (claude-sonnet-4-20250514)
- PWA: manifest.json + service worker

## 설계 원칙 (절대 원칙)

### 역할 분리
- 룰 엔진(코드): 주사위, 판정, HP/SAN 수치 계산, 사망 판정 전담
- AI 키퍼(Claude API): 묘사, 선택지 제시, 스토리텔링 전담
- AI는 수치를 절대 건드리지 않는다
- 판정 결과는 코드가 계산한 뒤 AI에 전달한다

### 시나리오 구조
- 시나리오는 JSON 데이터로 분리, 하드코딩 금지
- 엔진은 데이터만 읽어서 작동
- 추후 시나리오 편집기 UI를 붙일 수 있도록 확장 가능하게 설계

## CoC 7판 룰 구현 범위

### 캐릭터 시트
```
능력치 8개 (각 3d6×5로 생성):
STR(근력), CON(체질), SIZ(크기), DEX(민첩),
APP(외모), INT(지능), POW(권력), EDU(교육)

파생수치:
HP    = floor((CON + SIZ) / 10)
MP    = floor(POW / 5)
SAN   = POW × 5  (최대 99)
LUCK  = 3d6 × 5  (별도 굴림)
DB    = STR+SIZ 합계로 계산 (아래 표 참고)
BUILD = STR+SIZ 합계로 계산

DB/BUILD 계산표:
STR+SIZ  2~64   → DB: -2,   BUILD: -2
STR+SIZ  65~84  → DB: 0,    BUILD: -1
STR+SIZ  85~124 → DB: +1d4, BUILD: 0
STR+SIZ  125~164→ DB: +1d6, BUILD: 1
STR+SIZ  165~204→ DB: +2d6, BUILD: 2

기술치:
직업 포인트 = EDU × 4
개인관심 포인트 = INT × 2
각 기술에 포인트 배분, 기술치 상한 90%

나이 보정:
20~39세 보정 없음 (1단계에서 이 범위만 지원)
```

### 기본 기술 목록 및 기본값
```
회피:       DEX × 2
발견:       25%
도서관사용: 20%
심리학:     10%
은신:       20%
언변:       15%
응급처치:   30%
근접전투:   25%
권총:       20%
크툴루신화: 0%  ← 올라갈수록 SAN 최대치 감소
```

### 판정 시스템
```
d100 굴림 (1~100)

판정 등급:
대성공(Critical): roll == 1, 또는 roll <= floor(기술치/5)
어려운 성공(Hard): roll <= floor(기술치/2)
보통 성공(Regular): roll <= 기술치
대실패(Fumble): 기술치 < 50이면 roll >= 96, 기술치 >= 50이면 roll == 100
실패(Fail): 그 외

보너스 주사위: d100 두 번 굴려 낮은 값 선택 (유리한 상황)
패널티 주사위: d100 두 번 굴려 높은 값 선택 (불리한 상황)

행운(LUCK) 사용:
판정 실패 후 LUCK 소비 가능 → LUCK 수치만큼 굴려 성공 시 결과 변경
소비한 만큼 LUCK 수치 영구 감소
```

### SAN 시스템
```
SAN 체크:
d100 굴려서 현재 SAN 이하 = 성공
성공/실패에 따라 다른 손실량 적용 (이벤트별로 JSON에 정의)

광기 판정:
일시적 광기: 한 번의 SAN 체크에서 5 이상 손실 시 발동 → 1d10분 지속
부정기 광기: 세션 시작 SAN의 1/5 이상 누적 손실 시 발동
영구 광기: SAN = 0 → 게임오버

SAN 최대치:
max_SAN = min(99, 99 - 크툴루신화 기술치)
```

### 전투 시스템
```
라운드 구조:
1. DEX 높은 순서로 행동
2. 각 차례에 공격 OR 회피 택일 (동시 불가)

공격:
무기 기술로 판정
대성공 → 최대 피해 + DB
어려운 성공 → 피해 굴림 + DB, 상대 회피 불가
보통 성공 → 피해 굴림 + DB, 상대 회피 가능
실패/대실패 → 피해 없음

회피:
회피 기술로 판정
성공 시 피해 무효

피해 적용:
HP 0 이하 → 사망 → 게임오버 화면
HP 최대치의 절반 이하 → 중상 → CON 판정 (실패 시 기절)

기본 무기:
주먹: 근접전투, 1d3+DB, 최대피해 3+DB
칼:   근접전투, 1d4+2+DB, 최대피해 6+DB
권총: 권총, 1d10, 최대피해 10 (DB 미적용)
```

## AI 키퍼 시스템 프롬프트 방향

### 핵심 원칙
AI 키퍼는 스토리텔러이자 세계의 반응자다.
수치 계산은 절대 하지 않는다. 코드 엔진이 계산한 결과를 받아서 묘사만 한다.

### 분위기 지침
```
1. 공허를 공허로 남길 것
   → 모든 것을 설명하지 않는다
   → 미지의 것은 미지인 채로 묘사한다
   → "알 수 없는 무언가" 를 구체화하지 않는다

2. 편린만 보여줄 것
   → 존재의 전체를 드러내지 않는다
   → 윤곽, 그림자, 잔향만 묘사한다
   → 나머지는 플레이어의 상상에 맡긴다

3. 점진적 변모를 묘사할 것
   → 크툴루신화 수치와 SAN 수치를 참고한다
   → 초반: 평범한 탐정 묘사
   → 중반: 일상이 공허하게 느껴지기 시작
   → 후반: 진실을 아는 자와 모르는 자의 간극
   → 말기: 이전의 자아가 희미해짐

4. 인간 중심적이지 않은 세계
   → 우주적 존재는 인간에게 관심이 없다
   → 공포의 근원은 적대감이 아니라 무관심과 스케일
   → 인간의 판단과 행동이 우주적 맥락에서 무의미함을 암시

5. 가혹하게, 봐주지 않을 것
   → 판정 실패는 결과로 직결된다
   → 플레이어를 살리기 위한 임의 개입 금지
   → 사망은 사망이다
```

### 응답 형식 (JSON)
```json
{
  "narrative": "장면 묘사 (공허하고 이질적인 분위기)",
  "choices": ["행동1", "행동2", "행동3"],
  "requires_check": {
    "needed": true,
    "skill": "발견",
    "difficulty": "normal",
    "reason": "판정 요구 이유"
  },
  "san_check": {
    "needed": false,
    "loss": { "success": "0", "fail": "1d3" },
    "reason": ""
  },
  "combat_start": false

```

### 키퍼가 할 것 / 하지 말 것
```
할 것:
- 판정 결과를 받아서 그에 맞는 묘사
- 선택지 2~4개 제시 (자유 입력도 항상 허용)
- SAN 체크가 필요한 장면 신호
- 분위기와 긴장감 유지

하지 말 것:
- 수치 직접 계산 또는 변경
- 판정 없이 결과 결정
- 과도한 친절 또는 플레이어 유도
- 미지의 존재를 지나치게 구체적으로 묘사
- 해피엔딩 유도
```

## 게임 흐름

```
[캐릭터 생성]
→ 능력치 굴림 → 직업 선택 → 기술 포인트 배분

[탐색 페이즈]
플레이어 행동 입력
→ 판정 필요 여부 판단 (코드)
→ 필요 시 주사위 굴림 (코드)
→ 결과 + 현재 캐릭터 상태를 AI 키퍼에 전달
→ AI 키퍼 묘사 + 선택지 반환
→ SAN 체크 필요 시 → 굴림 → 손실 적용 → 광기 판정
→ 상태 저장 (localStorage)

[전투 페이즈]
DEX 순서 결정
→ 행동 선택 (공격/회피/도주)
→ 판정 굴림 (코드)
→ 피해 계산 (코드)
→ AI 키퍼 묘사
→ HP 0 시 사망 → 게임오버

[게임오버]
사망 or SAN 0
→ 결과 화면 (생존 시간, 최저 SAN, 발견한 단서 수, 크툴루신화 수치)
→ 다시 시작 or 새 캐릭터
```
